# Incident Drill Runbook

## Purpose

Run non-production incident drills to validate operator workflows, alert routing,
and evidence capture for SLO operations.

## Drill Scenario

- Example: checkout API latency regression with elevated error budget burn.

## Procedure

- Trigger synthetic symptom and burn-rate alert conditions.
- Verify routing to on-call and chat channels.
- Capture dashboard screenshots and response timeline.
- Record follow-up actions and owners.

## Success Criteria

- Drill completes in non-production.
- Evidence artifacts are captured and archived.
- At least one follow-up action is documented.
