## Overview

API: Tenant Control Plane API, version 1.0.0 (OpenAPI 3.1.0).

Control-plane API for tenant management (TR-21). The tenant contract document
validated by contracts/tenancy/TENANT_CONTRACT_SCHEMA_V1.json is the
authoritative tenant record; this API creates, reads, updates, and transitions
those documents. Lifecycle transitions execute the state machine of
contracts/tenancy/TENANT_LIFECYCLE_CONTRACT_V1.yaml verbatim and are
GitOps-only - a transition renders per-tenant overlays and isolation artifacts
per contracts/tenancy/TENANT_OVERLAY_GENERATION_CONTRACT_V1.yaml and
contracts/tenancy/TENANT_ISOLATION_MATRIX_V1.yaml and emits a prepared commit
reference; the service never writes mutable cluster or store state directly for
persistent configuration. This API is control-plane only: it carries tenant
management data and never tenant telemetry payloads (TR-16 plane separation).

## Schemas

### TenantDocument

The authoritative tenant contract document. Its lifecycle_state field is the
single source of truth for the state machine.

Type: `TENANT_CONTRACT_SCHEMA_V1.json`.

### LifecycleState

Exactly the lifecycle_state enum of
contracts/tenancy/TENANT_CONTRACT_SCHEMA_V1.json; no additional states are
permitted.

Type: string (enum); one of `provisioning`, `active`, `suspended`,
`offboarding`, `purged`.

### TenantUpdateRequest

Full tenant contract document with updated mutable fields; the result must
validate against contracts/tenancy/TENANT_CONTRACT_SCHEMA_V1.json. The fields
listed in x-immutable-fields must be byte-identical to the stored document or
the update is rejected with error_code update-forbidden-field.

Type: object.

### ApprovalRecordHighRisk

Approval record for risk class write.high-risk; required fields are exactly the
required_approval_fields of contracts/policy/APPROVAL_FLOW_V1.yaml. The decision
must be an approval; pending timeout and escalation semantics are owned by the
approval flow contract.

Type: object; no additional properties.

### GitOpsRenderReference

GitOps materialization of a transition (TR-10). Transitions change rendered
files and produce a prepared commit; they never mutate clusters or stores
directly for persistent configuration. Overlay paths and behavior per lifecycle
state (retain on suspend, retain until purge evidence during offboarding, remove
on purge) are owned by
contracts/tenancy/TENANT_OVERLAY_GENERATION_CONTRACT_V1.yaml.

Type: object; no additional properties.

| Field | Type | Required | Constraints | Description |
| ---- | ---- | ---- | ---- | ---- |
| `render_action` | string (enum) | yes | one of `rendered`, `retained`, `removed`, `not-applicable`, `replayed-no-diff` | rendered: overlay (re)generated. retained: overlay kept unchanged by contract (for example suspend and offboard). removed: overlay directory removed via git commit (purge). not-applicable: no overlay exists for the isolation class. replayed-no-diff: idempotent replay produced no diff and no commit. |
| `overlay_path` | string | no |  | Repository path of the tenant overlay directory (gitops/overlays/tenants/<tenant_id>/), when one applies. |
| `commit_ref` | string | no |  | Opaque reference to the prepared or recorded git commit carrying the render. Absent when render_action is not-applicable or replayed-no-diff. |

### AuditRecordRef

Reference to the audit record emitted for a transition attempt (TR-09). Every
transition response and every transition denial carries one; required fields per
transition are bound in
`x-lifecycle-binding.transitions.<name>.audit_required_fields` and restated on
each response schema. Audit records are control-plane records and never embed
tenant telemetry payloads.

Type: object.

| Field | Type | Required | Constraints | Description |
| ---- | ---- | ---- | ---- | ---- |
| `audit_record_id` | string | yes | min length 1 |  |
| `tenant_id` | string | yes |  |  |
| `transition` | string (enum) | yes | one of `provision`, `suspend`, `resume`, `offboard`, `purge` |  |
| `actor` | string | yes |  |  |
| `replay` | boolean | yes |  | True when this attempt was an idempotent replay of a completed transition. |
