# IMPLEMENTATION BATCH COMMAND SHEET PROMPT

## REFERENCES

- @docs/auxiliary/planning/IMPLEMENTATION_TASKS.md
- @docs/auxiliary/task_execution/BATCH_COMMAND_SHEET.md

## TASK

Create a new Markdown document named `IMPLEMENTATION_BATCH_COMMAND_SHEET.md` based
on `IMPLEMENTATION_TASKS.md`, using the attached `BATCH_COMMAND_SHEET.md` as the
style and structure reference.

## REQUIREMENTS

- Keep it short, clear, and execution-oriented.
- Preserve the same overall format and usability pattern as the reference sheet.
- Create one command row per implementation batch in `IMPLEMENTATION_TASKS.md`.
- Each row must contain:
  - Batch identifier (supports numeric and alphanumeric IDs such as `9A`)
  - Exact command to send
  - Required validation step
- Include these sections:
  - Title
  - Short usage note
  - Batch Commands
  - Optional Strict Commands
  - Stop/Resume Commands
  - Example
- Commands must match the actual batch names and order from
  `IMPLEMENTATION_TASKS.md`.
- Validation steps must reflect the real completion and validation gates from that
  document.
- Do not invent extra batches.
- Do not output commentary.
- Output Markdown only.
